📋 RESUMO EXECUTIVO
═══════════════════════════════════════════════════════════════════════════════

O AI Service é um serviço microserviço completo em NestJS que fornece:

1. iMestreAI - Assistente conversacional inteligente
   → Chat com histórico de conversas
   → Processamento de comandos em linguagem natural
   → Integração com OpenAI GPT-4

2. Motor de Análise - Insights automáticos
   → Análise de vendas (tendências, sazonalidade)
   → Alertas de estoque (crítico, encalhado)
   → Análise financeira (fluxo de caixa)
   → Classificação fiscal (pendências)
   → Insights de marketplace

3. Previsão de Demanda
   → Algoritmo combinado (média móvel + regressão + sazonalidade)
   → Cálculo de ponto de reposição
   → Estimativa de confiança

4. Classificação Fiscal
   → Sugestão de NCM via IA
   → Sugestão de CFOP
   → Validação contra tabela TIPI

5. Sugestões Inteligentes
   → Preço (baseado em custo + mercado)
   → Descrição de produto (otimizada SEO)
   → Respostas para clientes (marketplace)
   → Títulos de anúncio (por marketplace)

🚀 QUICK START
═══════════════════════════════════════════════════════════════════════════════

1. Clonar e preparar:
   cd services/ai-service
   cp .env.example .env.development
   # Editar .env.development com OPENAI_API_KEY

2. Instalar dependências:
   npm install

3. Iniciar stack local:
   docker-compose up -d
   sleep 10

4. Setup Prisma:
   npm run prisma:generate
   npm run prisma:migrate

5. Iniciar servidor:
   npm run dev

6. Acessar Swagger:
   http://localhost:3008/docs

🆘 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Erro: OPENAI_API_KEY not configured
→ Configure em .env.development e reinicie

Erro: PostgreSQL connection refused
→ docker-compose ps (verificar se está rodando)
→ docker-compose logs postgres (ver logs)

Erro: Kafka broker unavailable
→ Aguarde inicialização do Redpanda (~30 segundos)
→ docker-compose logs redpanda

Erro: Rate limit exceeded
→ Aumente RATE_LIMIT_TOKENS_POR_MINUTO

Banco de dados vazio?
→ npm run prisma:migrate
