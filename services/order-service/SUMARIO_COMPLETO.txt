📊 FUNCIONALIDADES PRINCIPAIS

GERENCIAMENTO DE PEDIDOS
✅ Criar pedido (múltiplas origens: marketplace, ecommerce, loja física, manual)
✅ Máquina de estados: RASCUNHO → PENDENTE → CONFIRMADO → SEPARANDO → SEPARADO → FATURADO → ENVIADO → ENTREGUE
✅ Histórico auditável de todas as mudanças
✅ Cancelamento com liberação de estoque
✅ Validação de transições de status
✅ Listar com paginação e filtros (status, canal, data, cliente)
✅ Rastreamento com código e transportadora
✅ Cálculo automático de totais

INTEGRAÇÃO COM OUTROS SERVIÇOS
✅ fiscal-service: Faturamento com NF-e (publica PEDIDO_FATURAR, consome NOTA_AUTORIZADA)
✅ inventory-service: Reserva de estoque (consome ESTOQUE_RESERVADO/INSUFICIENTE)
✅ payment-service: Pagamentos (consome PAGAMENTO_AUTORIZADO/CAPTURADO/RECUSADO)
✅ marketplace-service: Sincronização (consome MARKETPLACE_PEDIDO_RECEBIDO)

GESTÃO DE PAGAMENTOS
✅ Registrar pagamento (6 tipos: cartão, boleto, PIX, transferência, marketplace)
✅ 5 status: PENDENTE → AUTORIZADO → PAGO (ou ESTORNADO/REEMBOLSADO)
✅ Suporte a parcelas
✅ Webhooks de gateway
✅ Estorno de pagamento

GESTÃO DE DEVOLUÇÕES
✅ Solicitar devolução (SOLICITADA)
✅ Aprovar com código de retorno (APROVADA)
✅ Receber devolução (RECEBIDA) com reintegração de estoque
✅ Reembolsar (REEMBOLSADA)
✅ Devolução por item específico

CACHE E PERFORMANCE
✅ Redis para pedidos individuais (TTL 10 min)
✅ Invalidação automática de listas
✅ Padrões de chaves por tenant
✅ Scan eficiente para grandes datasets

MULTI-TENANCY

🚀 COMO INICIAR
───────────────────────────────────────────────────────────────────────────────

1. SETUP
   npm install
   npm run db:migrate
   npm run db:seed

2. DESENVOLVIMENTO
   npm run dev
   → http://localhost:3005/api/docs

3. TESTES
   npm run test:unit
   npm run test:integration
   npm run test:e2e
   npm run test:cov

4. BUILD & DEPLOY
   npm run build
   npm start
   
   # Ou Docker
   docker build -t imestredigital/order-service .
   docker run -p 3005:3005 imestredigital/order-service
